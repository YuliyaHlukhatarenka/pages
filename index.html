Definition

REDUX is a container for managing state of application.

State is an regular JavaScript object

What is a state? State can include locally created data, server responses, UI state of your application, and so on.

State is an regular JavaScript object. It is stored in one place: in the Redux Store (storage). The example of Store for some game application.

Managing state is hard sometime. If a model can update another model, then a view can update a model, which updates another model, and this, in turn, might cause another view to update. At some point, you no longer understand what happens in your app as you have lost control over the when, why, and how of its state.

How does it work?

View dispatchs an action when some event happens in View. Action is passed to reducer.

Reducer changes the Store state. Components that are subscribed to the Store updates get notification.

This is typical architecture of React app. Data goes down to the child Components through properties.The child components usr functions(are pased by parent) to inform about state changes. Redux is technology that separate state form user interface. Each component can interfere withstore directly.

Core Concepts

  • store
  • actions and action creators

Store

How to create Store:

STORE it is a state of application. It is read only.

Picture shows how to add Store to you application. When we create a Store we have to pass the reducer as a parameter.

What we can do with the Store:

  • get current state
  • subscribe listeners to change events
  • update state by reducer

To get current state use:

store.getState()

To subscribe listeners use:

store.suscribe()

This code shows how to subrcrib to store changes and get current state. Unregistering of listeners you can implement by function that is returned by subscribe method. const unsubscribe = store.subscribe(listener).

To update state by reducer use:

store.dispatch(action)

Example shows the code for form that add new fruit to the array (Store). Action = "addFruit".

Action

Action is an object

{ type: ADD, payload: text }

Action is payload of information that send data from your component to your store. Actions is an object. Usualy the first property is Action type. It is good practice to define type as string constant. The second property is payload (information to change the Store). You send them to the store using store.dispatch().

Action creators

Action creator is a function that create action. It returns an action object.

Reducers

Reducers specify how the application's state changes in response to actions sent to the store. Actions only describe what happened and reducer describe how the application's state changes.
